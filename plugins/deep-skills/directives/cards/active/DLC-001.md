# DLC-001 — State / Data-Flow Contract

**Why this card exists.** Every severity-bearing finding in the PR#65 benchmark (CR-001 Blocker → CR-007) was one bug family: *a write path whose runtime reader resolves a different source* ("execution is not config-aware"). None were in-function logic errors — they were integration/contract bugs at phase boundaries, which is why code-review caught them late and expensively. This card pushes the kill upstream. Hand-authored from PR#65; would have surfaced **~100% of that review's severity at plan time.**

Each spoke applies only its own section below.

---

## `deep-plan` — add the contract artifact (required plan section)

The plan is **not "done"** while any contract row is unresolved. Add this section to the plan document:

```markdown
## State / Data-Flow Contract

| # | State (field / table / store) | Writer(s) path:line | Runtime reader(s) path:line | Same source? | Resolution / AC |
|---|---|---|---|---|---|
```

**One row per piece of state the feature reads or writes at runtime.** A row is *resolved* only if the runtime reader resolves the **same source** the writer wrote to. The row cannot be filled honestly without discovering the bug: the writer column forces "where does the save land?", the reader column forces a runtime trace, and "Same?" is a binary you cannot fudge.

**The four rules that give it teeth:**
1. Every new persisted column / store / localStorage key the plan introduces **must** appear as a writer row.
2. Every "single source of truth" claim in any AC must point to **exactly one** source. Two writers feeding one concept = a conflict row to resolve.
3. **Deletion-parity:** when the plan removes a surface, each behavior it provided becomes a row whose reader must be re-homed.
4. Reader = "none", or reader-source ≠ writer-source ⇒ **blocks plan completion** unless explicitly marked Display-only and moved to Deferreds.

## `deep-plan-review` — dispatch check

Add to the review dispatch: assert the **State / Data-Flow Contract** section is present, **every row is resolved** (reader resolves the writer's source), and every "single source of truth" AC names **exactly one** source. An unresolved row that is not explicitly Display-only + Deferred is a review finding that blocks the plan.

## `deep-implement` — validate (exit invariant + contract probe)

Exit invariant: for each "runtime" contract row, the reader is actually wired in code to the writer's source. Then run a **contract probe** for it — an end-to-end check of the form **write → execute → assert read** (write via the real surface, execute the runtime path, assert the executed behavior reflects the written value). Typecheck + scoped unit tests + prompt snapshot would have caught **none** of CR-001–006 (they typecheck clean and pass unit tests; the bug is cross-layer). The contract probe is the missing net.
